# Open API 外部前端接入

本文档面向需要接入 `fun-ai-claw-api` 的外部前端系统。

## 目标与边界

当前系统存在两套接入层：

- 控制台内部使用：`/v1/**`
- 外部系统使用：`/open/v1/**`

本次新增的外部接入层不会影响现有控制台逻辑，因此：

- 现有控制台前端继续走 `/v1/**`
- 第三方前端统一走 `/open/v1/**`
- 不建议外部系统直接依赖控制台私有接口

## 接入流程

推荐接入顺序如下：

1. 后台为外部系统分配一个 `open_client_app`
2. 外部系统调用 `POST /open/v1/sessions` 创建或恢复会话
3. 接口返回：
   - `sessionId`
   - `websocketPath`
   - `websocketToken`
4. 外部前端使用 `sessionId + websocketToken` 建立 WebSocket 连接
5. 建连后通过 WebSocket 发送用户消息
6. 前端接收流式事件：`delta`、`message`、`debug`
7. 如需回放历史，调用 `GET /open/v1/sessions/{sessionId}/messages`
8. 会话结束时，调用 `POST /open/v1/sessions/{sessionId}/close`

## 鉴权方式

### REST 鉴权

`/open/v1/**` 默认使用以下请求头：

- `X-Open-App-Id`
- `X-Open-App-Secret`

默认配置位于 API 服务的 `application.yml`：

```yaml
app:
  open:
    auth:
      app-id-header: X-Open-App-Id
      app-secret-header: X-Open-App-Secret
```

### WebSocket 鉴权

浏览器环境下不适合依赖自定义鉴权头，因此 WebSocket 改为使用会话令牌：

- `sessionId`
- `wsToken`

其中 `wsToken` 由 `POST /open/v1/sessions` 返回。

## 数据准备

### 新增表

外部接入层新增了以下表：

- `open_client_app`
- `open_session`
- `open_session_message`

建表语句已同步到 API 项目的 `schema.sql`。

### 初始化一个外部应用

你需要先在数据库中插入一条 `open_client_app` 记录。

建议 `app_secret_hash` 使用：

- `sha256:<hex>`

当前实现也兼容直接明文匹配，但线上不建议这么做。

示例：

```sql
insert into open_client_app (
    app_id,
    name,
    app_secret_hash,
    enabled,
    default_instance_id,
    default_agent_id,
    created_at,
    updated_at
) values (
    'partner-web',
    'Partner Web',
    'sha256:PUT_SHA256_HEX_HERE',
    true,
    '00000000-0000-0000-0000-000000000000',
    'mgc-novel-to-script',
    now(),
    now()
);
```

## REST 接口

### 1. 创建或恢复会话

接口：

- `POST /open/v1/sessions`

请求头：

```http
X-Open-App-Id: partner-web
X-Open-App-Secret: your-secret
Content-Type: application/json
```

请求体示例：

```json
{
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "agentId": "mgc-novel-to-script",
  "externalUserId": "user-1001",
  "externalSessionKey": "order-20260310-001",
  "title": "小说转剧本对话",
  "metadata": {
    "channel": "h5",
    "traceId": "trace-001"
  }
}
```

字段说明：

- `instanceId`：可选，未传时回退到 `open_client_app.default_instance_id`
- `agentId`：可选，未传时回退到 `open_client_app.default_agent_id`
- `externalSessionKey`：建议传，相同 `appId + externalSessionKey` 会复用已有会话，并重新签发新的 `wsToken`
- `metadata`：原样保存为 JSON 文本

成功响应示例：

```json
{
  "sessionId": "11111111-1111-1111-1111-111111111111",
  "appId": "partner-web",
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "agentId": "mgc-novel-to-script",
  "externalUserId": "user-1001",
  "externalSessionKey": "order-20260310-001",
  "title": "小说转剧本对话",
  "status": "ACTIVE",
  "metadata": {
    "channel": "h5",
    "traceId": "trace-001"
  },
  "websocketPath": "/open/v1/agent-session/ws?sessionId=11111111-1111-1111-1111-111111111111&wsToken=...",
  "websocketToken": "...",
  "websocketTokenExpiresAt": "2026-03-11T03:00:00Z",
  "createdAt": "2026-03-10T03:00:00Z",
  "updatedAt": "2026-03-10T03:00:00Z",
  "lastMessageAt": null
}
```

### 2. 查询会话

接口：

- `GET /open/v1/sessions/{sessionId}`

用途：

- 查询当前会话元信息
- 页面刷新后恢复会话信息

### 3. 查询历史消息

接口：

- `GET /open/v1/sessions/{sessionId}/messages?limit=100`

成功响应结构：

```json
{
  "items": [
    {
      "id": "22222222-2222-2222-2222-222222222222",
      "sessionId": "11111111-1111-1111-1111-111111111111",
      "eventType": "user_input",
      "role": "user",
      "content": "请帮我生成一句话剧本",
      "thinkingContent": null,
      "interaction": null,
      "providerMessageId": null,
      "providerSequence": null,
      "pending": false,
      "emittedAt": "2026-03-10T03:01:00Z",
      "createdAt": "2026-03-10T03:01:00Z",
      "rawPayload": "请帮我生成一句话剧本"
    }
  ]
}
```

说明：

- `limit` 最大 `500`
- 当前会保存：
  - 用户发送给 WebSocket 的原始输入
  - agent 返回的最终 `message` 帧
- 当前不会落库：
  - `delta`
  - `debug`

### 4. 关闭会话

接口：

- `POST /open/v1/sessions/{sessionId}/close`

关闭后该 session 状态会变为 `CLOSED`。

## WebSocket 接口

### 建连地址

接口：

- `WS /open/v1/agent-session/ws?sessionId=<uuid>&wsToken=<token>`

说明：

- 这是外部前端专用的 WebSocket 入口
- API 会校验 `sessionId + wsToken`
- 校验通过后，再转发到 plane 内部的 `agent-session`

### 发送消息

WebSocket 建连后，前端直接发送纯文本即可。

示例：

```text
script_type=一句话剧本
script_content=一个被陷害的女律师重生回庭审前夜，反杀幕后黑手并逆袭成律所合伙人。
target_audience=女频
expected_episode_count=5
```

如果是交互确认，也沿用当前 `agent-session` 的文本协议：

```text
interaction_action=confirm
stateId=step1_input_parse
```

### 接收事件

当前 WebSocket 会透传现有 `agent-session` 的事件帧，主要包括：

- `message`
- `delta`
- `debug`

`delta` 用于流式正文和 thinking 渲染，`message` 用于最终消息和交互块。

## 前端接入示例

### 创建会话

```ts
const sessionResp = await fetch('/open/v1/sessions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Open-App-Id': 'partner-web',
    'X-Open-App-Secret': 'your-secret',
  },
  body: JSON.stringify({
    instanceId: '00000000-0000-0000-0000-000000000000',
    agentId: 'mgc-novel-to-script',
    externalUserId: 'user-1001',
    externalSessionKey: 'biz-session-001',
    title: '小说转剧本',
  }),
});

const session = await sessionResp.json();
```

### 建立 WebSocket

```ts
const wsBase = location.origin.replace(/^http/i, 'ws');
const socket = new WebSocket(`${wsBase}${session.websocketPath}`);

socket.onopen = () => {
  socket.send([
    'script_type=一句话剧本',
    'script_content=一个被陷害的女律师重生回庭审前夜，反杀幕后黑手并逆袭成律所合伙人。',
    'target_audience=女频',
    'expected_episode_count=5',
  ].join('\n'));
};

socket.onmessage = (event) => {
  const frame = JSON.parse(event.data);
  if (frame.eventType === 'delta') {
    return;
  }
  if (frame.eventType === 'message') {
    return;
  }
  if (frame.eventType === 'debug') {
    return;
  }
};
```

## 当前实现限制

当前 `/open/v1` 已可用于外部前端接入，但仍有以下限制：

1. 暂未提供纯 REST 的“发送消息”接口，当前消息收发必须走 WebSocket
2. 历史消息当前只落：
   - 用户输入
   - 最终 `message` 帧
3. `delta` 与 `debug` 当前只做实时透传，不落库
4. 暂未提供外部应用管理接口，`open_client_app` 需要后台或 DBA 手动维护
5. 暂未加签名、限流、IP 白名单等更强安全控制
