# 用户中心 syncUserInfo 接口

本文档面向外部用户中心或统一账号系统，用于在用户注册成功、资料更新或首次登录后，将用户基础资料同步到 funClaw 本地数据库。

说明：

- 该接口的目标是同步 `consumer_account`
- 该接口不负责聊天、实例绑定或积分扣减
- 该接口当前由 funClaw API 提供，不属于 `/open/v1/**`
- 对外推荐将该接口名称统一称为 `syncUserInfo`

## 基础信息

- Base URL：`/integration/v1`
- 接口路径：`POST /integration/v1/consumer/accounts/syncUserInfo`
- Content-Type：`application/json`

## 推荐时序

推荐由外部用户中心服务端在“注册成功 / 登录成功”前完成 `syncUserInfo` 调用。

```text
用户前端            外部用户中心前端        外部用户中心服务端            funClaw API
   |                      |                        |                           |
   | 输入手机号/验证码      |                        |                           |
   |--------------------->|                        |                           |
   |                      | 提交登录/注册请求        |                           |
   |                      |----------------------->|                           |
   |                      |                        | 校验验证码/创建登录态        |
   |                      |                        |-------------------------->|
   |                      |                        |<--------------------------|
   |                      |                        |                           |
   |                      |                        | 调用 syncUserInfo          |
   |                      |                        |-------------------------->|
   |                      |                        |<--------------------------|
   |                      |                        |                           |
   |                      |                        | 同步成功，返回登录成功       |
   |                      |<-----------------------|                           |
   |<---------------------|                        |                           |
```

### 成功判定建议

- 只有当外部用户中心自己的登录/注册逻辑成功，且 `syncUserInfo` 返回成功时，才向用户前端返回“登录成功”或“注册成功”
- 如果用户中心本身登录成功，但 `syncUserInfo` 失败，建议整体视为失败或进入可重试状态
- 不建议把 `syncUserInfo` 放到浏览器侧调用，因为同步密钥不应暴露给前端

## 鉴权

默认使用请求头：

```http
X-Open-App-Id: your-app-id
X-Consumer-Sync-Secret: your-sync-secret
Content-Type: application/json
```

其中：

- `CONSUMER_AUTH_SYNC_ENABLED`
- `CONSUMER_AUTH_SYNC_SECRET_HEADER`
- `APP_OPEN_AUTH_APP_ID_HEADER`（默认 `X-Open-App-Id`）

说明：

- 当 `CONSUMER_AUTH_SYNC_ENABLED=false` 时，接口不可用
- `X-Open-App-Id` 对应开放平台中的应用 `app_id`
- `X-Consumer-Sync-Secret` 对应开放平台中该应用配置的 `app_secret`
- 服务端会根据 `app_id` 查询已启用的开放平台应用，再校验 `app_secret`
- 如果希望使用 `Authorization` 头，也可以把 `CONSUMER_AUTH_SYNC_SECRET_HEADER` 配成 `Authorization`

## 适用场景

- 用户在外部用户中心注册成功后，同步一份本地账户
- 用户首次登录 funClaw 前，由用户中心预先推送用户资料
- 用户昵称、头像、手机号等资料发生变化后，增量覆盖本地快照

## 请求体

请求体字段按当前外部用户中心的 `userInfo` 风格设计：

```json
{
  "id": "12345",
  "uid": "u_12345",
  "userName": "张三",
  "avatar": "https://example.com/avatar.png",
  "invitationCode": "ABCD1234",
  "phone": "13800138000",
  "payUserId": "998877",
  "lastLogin": "2026-03-18T10:30:00Z",
  "status": "1",
  "type": "0",
  "createdAt": "2026-03-01T08:00:00Z"
}
```

### 字段说明

- `id`：必填；外部用户唯一 ID，会写入本地 `external_user_id`
- `uid`：可选；外部系统侧的辅助 UID
- `userName`：可选；用户昵称
- `avatar`：可选；头像 URL
- `invitationCode`：可选；邀请码，当前只做原样接收，不参与本地账户主逻辑
- `phone`：可选；手机号，当前会同时写入 `phone_e164` 与 `phone_masked`
- `payUserId`：可选；支付系统用户 ID，当 `uid` 为空时可作为 `external_uid` 的回退值
- `lastLogin`：可选；最近登录时间，建议传 ISO-8601 时间字符串
- `status`：可选；当前已兼容 `1 -> ACTIVE`、`0 -> DISABLED`
- `type`：可选；当前只接收，不参与本地账户主逻辑
- `createdAt`：可选；外部账户创建时间，建议传 ISO-8601 时间字符串
- `sourceSystem`：当前无需传；本地 `consumer_account.source_system` 固定写入 `user-center`

## 字段映射

同步到本地 `consumer_account` 时，核心映射如下：

- `id` → `external_user_id`
- `uid / payUserId / id` → `external_uid`
- `userName` → `display_name`
- `avatar` → `avatar_url`
- `phone` → `phone_e164`
- `phone` → `phone_masked`
- `status` → `status`
- `createdAt` → `external_created_at`
- `lastLogin` → `last_login_at`

## 响应示例

```json
{
  "created": true,
  "account": {
    "accountId": "11111111-1111-1111-1111-111111111111",
    "externalUserId": "12345",
    "externalUid": "u_12345",
    "sourceSystem": "user-center",
    "phoneE164": "13800138000",
    "phoneMasked": "13800138000",
    "displayName": "张三",
    "avatarUrl": "https://example.com/avatar.png",
    "status": "ACTIVE",
    "externalCreatedAt": "2026-03-01T08:00:00Z",
    "linkedAt": "2026-03-18T10:31:00Z",
    "updatedAt": "2026-03-18T10:31:00Z",
    "lastLoginAt": "2026-03-18T10:30:00Z",
    "lastVerifiedAt": "2026-03-18T10:31:00Z",
    "activeSessionCount": 0,
    "activeInstanceCount": 0
  }
}
```

### 响应字段说明

- `created=true`：表示本次同步新建了本地账户
- `created=false`：表示本次同步命中了已有账户，并更新了资料快照
- `account`：返回 funClaw 当前保存的本地账户视图

## 幂等性

该接口按 `external_user_id` 幂等：

- 同一个 `id` 第一次同步时会创建本地账户
- 后续再次同步相同 `id` 时会更新该用户的快照数据
- 不会重复创建多条本地账户记录

## 调用示例

### cURL

```bash
curl -X POST "http://your-host/integration/v1/consumer/accounts/syncUserInfo" \
  -H "Content-Type: application/json" \
  -H "X-Open-App-Id: your-app-id" \
  -H "X-Consumer-Sync-Secret: your-sync-secret" \
  -d '{
    "id": "12345",
    "uid": "u_12345",
    "userName": "张三",
    "avatar": "https://example.com/avatar.png",
    "phone": "13800138000",
    "payUserId": "998877",
    "lastLogin": "2026-03-18T10:30:00Z",
    "status": "1",
    "createdAt": "2026-03-01T08:00:00Z"
  }'
```

### 典型流程

1. 外部用户中心完成注册或登录
2. 外部系统拿到用户资料
3. 调用本接口把用户资料同步到 funClaw
4. funClaw 后续基于本地 `consumer_account` 完成实例绑定、会话归属等业务

## 错误说明

- `401 Unauthorized`：`X-Open-App-Id` 或同步密钥错误、未传，或对应开放平台应用未启用
- `404 Not Found`：`syncUserInfo` 接口已被关闭
- `400 Bad Request`：请求体缺少 `id` 等关键字段

## 当前限制

- 当前仅同步用户基础资料，不直接绑定实例
- 当前不处理积分、支付、邀请码核销等业务
- `phone` 目前直接写入本地手机号字段，未额外做脱敏
- `createdAt` / `lastLogin` 当前推荐使用 ISO-8601；格式不合法时会被忽略
- 本地 `consumer_account.source_system` 当前固定为 `user-center`
