# Claw 实例与 Agent 模型

## 领域模型

- `Host`：运行 Docker 的物理或虚拟机
- `ClawInstance`：由执行面管理的隔离运行单元
- `Agent`：Claw 实例内部的逻辑工作者
- `Task`：分配给 Agent 的可执行工作单元

## 映射策略

- 一个 `ClawInstance` 可以承载多个 Agent
- Agent 应当暴露稳定的元数据：
  - `agentId`
  - `agentType`
  - `version`
  - `capabilities`
- 实例级生命周期操作（`START`、`STOP`、`RESTART`、`ROLLBACK`）保留在控制 API
- Agent 级行为文档与运维手册放在本文档站

## 为什么这样拆分

- **控制台**：实时操作与动作执行
- **文档站**：持久化知识、标准规范与故障排查上下文
