# Open API 外部前端接入

本文档面向第三方系统开发者，只描述 `/open/v1/**` 的对外契约，不包含控制台内部实现细节。

## 基础信息

- REST Base: `/open/v1`
- WebSocket: `/open/v1/agent-session/ws`
- 会话模式：先创建会话，再通过 WebSocket 收发消息

## 会话模式说明

`/open/v1` 同时支持以下两种模式：

- 指定 Agent 模式：创建会话时传入 `agentId`，或外部应用已配置 `default_agent_id`
- 自动路由模式：创建会话时不传 `agentId`，且外部应用也未配置 `default_agent_id`

建议：

- 已明确目标 Agent 时，优先使用指定 Agent 模式
- 需要由系统自行判断入口 Agent 时，使用自动路由模式

## 鉴权

### REST 鉴权

所有 REST 接口都需要以下请求头：

```http
X-Open-App-Id: your-app-id
X-Open-App-Secret: your-app-secret
Content-Type: application/json
```

### WebSocket 鉴权

WebSocket 不使用自定义请求头，而是使用创建会话时返回的：

- `sessionId`
- `wsToken`

连接格式：

```text
/open/v1/agent-session/ws?sessionId=<uuid>&wsToken=<token>
```

## 接入流程

1. 获取平台分配的 `appId` 和 `appSecret`
2. 调用 `POST /open/v1/sessions` 创建或恢复会话
3. 使用返回的 `sessionId` 和 `wsToken` 建立 WebSocket 连接
4. 通过 WebSocket 发送用户消息并接收流式事件
5. 如需回放历史，调用 `GET /open/v1/sessions/{sessionId}/messages`
6. 会话结束时，调用 `POST /open/v1/sessions/{sessionId}/close`

## REST 接口

### 1. 创建或恢复会话

- 方法：`POST /open/v1/sessions`
- 用途：创建新会话，或按 `externalSessionKey` 恢复已有活动会话

请求体字段：

- `instanceId`: 可选；如果应用已配置默认实例，可以不传
- `agentId`: 可选；如果应用已配置默认 Agent，可以不传
- `externalSessionKey`: 推荐；同一 `appId + externalSessionKey` 会复用活动会话，并重新签发新的 `wsToken`

请求示例：

```json
{
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "agentId": "mgc-novel-to-script",
  "externalSessionKey": "biz-session-001"
}
```

响应示例：

```json
{
  "sessionId": "11111111-1111-1111-1111-111111111111",
  "appId": "partner-web",
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "agentId": "mgc-novel-to-script",
  "externalSessionKey": "biz-session-001",
  "status": "ACTIVE",
  "websocketPath": "/open/v1/agent-session/ws?sessionId=11111111-1111-1111-1111-111111111111&wsToken=...",
  "websocketToken": "...",
  "websocketTokenExpiresAt": "2026-03-11T03:00:00Z",
  "createdAt": "2026-03-10T03:00:00Z",
  "updatedAt": "2026-03-10T03:00:00Z",
  "lastMessageAt": null
}
```

说明：

- 如果没有传 `instanceId`，且应用也未配置默认实例，请求会失败
- `websocketToken` 只在创建或恢复会话时返回
- 后续 `GET /sessions/{sessionId}` 不会再次返回 `websocketToken`

### 2. 查询会话

- 方法：`GET /open/v1/sessions/{sessionId}`
- 用途：查询会话状态与基本信息

响应字段与创建会话基本一致，但 `websocketToken` 一般为空。

### 3. 查询历史消息

- 方法：`GET /open/v1/sessions/{sessionId}/messages?limit=100`
- 用途：获取当前会话的历史消息

查询参数：

- `limit`: 可选；默认 `100`，范围 `1-500`

响应示例：

```json
{
  "items": [
    {
      "id": "22222222-2222-2222-2222-222222222222",
      "sessionId": "11111111-1111-1111-1111-111111111111",
      "eventType": "message",
      "role": "assistant",
      "content": "# 输入解析\n...",
      "thinkingContent": null,
      "interaction": {
        "version": "1.0",
        "type": "approval_request"
      },
      "providerMessageId": "msg-001",
      "providerSequence": 1,
      "pending": false,
      "emittedAt": "2026-03-10T03:05:00Z",
      "createdAt": "2026-03-10T03:05:00Z",
      "rawPayload": "{...}"
    }
  ]
}
```

说明：

- 当前默认只落库用户输入和最终 `message` 事件
- 如果服务端关闭消息持久化，历史消息接口可能返回空列表
- 历史消息接口不保证包含实时 `delta` 增量

### 4. 关闭会话

- 方法：`POST /open/v1/sessions/{sessionId}/close`
- 用途：主动结束会话

响应仍返回会话对象，`status` 会变为 `CLOSED`。

## 调试示例

以下示例均基于最新契约，请求体里不再包含 `externalUserId`。

### cURL 示例

先准备环境变量：

```bash
BASE_URL=http://your-host/fun-claw/api/open/v1
APP_ID=your-app-id
APP_SECRET=your-app-secret
INSTANCE_ID=00000000-0000-0000-0000-000000000000
AGENT_ID=mgc-novel-to-script
EXTERNAL_SESSION_KEY=biz-session-001
SESSION_ID=11111111-1111-1111-1111-111111111111
```

创建或恢复会话：

```bash
curl -X POST "$BASE_URL/sessions" \
  -H "Content-Type: application/json" \
  -H "X-Open-App-Id: $APP_ID" \
  -H "X-Open-App-Secret: $APP_SECRET" \
  -d '{
    "instanceId": "'"$INSTANCE_ID"'",
    "agentId": "'"$AGENT_ID"'",
    "externalSessionKey": "'"$EXTERNAL_SESSION_KEY"'"
  }'
```

查询会话：

```bash
curl "$BASE_URL/sessions/$SESSION_ID" \
  -H "X-Open-App-Id: $APP_ID" \
  -H "X-Open-App-Secret: $APP_SECRET"
```

查询历史消息：

```bash
curl "$BASE_URL/sessions/$SESSION_ID/messages?limit=100" \
  -H "X-Open-App-Id: $APP_ID" \
  -H "X-Open-App-Secret: $APP_SECRET"
```

关闭会话：

```bash
curl -X POST "$BASE_URL/sessions/$SESSION_ID/close" \
  -H "X-Open-App-Id: $APP_ID" \
  -H "X-Open-App-Secret: $APP_SECRET"
```

如果你走自动路由模式，创建会话时直接去掉 `agentId` 即可。

### Postman 示例

建议先在 Postman 里配置环境变量：

- `baseUrl`: `http://your-host/fun-claw/api/open/v1`
- `appId`: 平台分配的 `appId`
- `appSecret`: 平台分配的 `appSecret`
- `instanceId`: 目标实例 ID
- `agentId`: `mgc-novel-to-script`
- `externalSessionKey`: 业务侧会话键
- `sessionId`: 创建会话后返回的 `sessionId`
- `wsToken`: 创建会话后返回的 `websocketToken`

创建或恢复会话：

- Method: `POST`
- URL: `{{baseUrl}}/sessions`
- Headers:
  - `Content-Type: application/json`
  - `X-Open-App-Id: {{appId}}`
  - `X-Open-App-Secret: {{appSecret}}`
- Body:

```json
{
  "instanceId": "{{instanceId}}",
  "agentId": "{{agentId}}",
  "externalSessionKey": "{{externalSessionKey}}"
}
```

查询会话：

- Method: `GET`
- URL: `{{baseUrl}}/sessions/{{sessionId}}`
- Headers:
  - `X-Open-App-Id: {{appId}}`
  - `X-Open-App-Secret: {{appSecret}}`

查询历史消息：

- Method: `GET`
- URL: `{{baseUrl}}/sessions/{{sessionId}}/messages?limit=100`
- Headers:
  - `X-Open-App-Id: {{appId}}`
  - `X-Open-App-Secret: {{appSecret}}`

关闭会话：

- Method: `POST`
- URL: `{{baseUrl}}/sessions/{{sessionId}}/close`
- Headers:
  - `X-Open-App-Id: {{appId}}`
  - `X-Open-App-Secret: {{appSecret}}`

如果要在 Postman 里测试 WebSocket，可新建 WebSocket Request，连接：

```text
ws://your-host/fun-claw/api/open/v1/agent-session/ws?sessionId={{sessionId}}&wsToken={{wsToken}}
```

发送内容使用纯文本，例如：

```text
script_type=一句话剧本
script_content=一个被陷害的女律师重生回庭审前夜，反杀幕后黑手并逆袭成律所合伙人。
target_audience=女频
expected_episode_count=5
```

## WebSocket 接口

### 1. 建立连接

```text
WS /open/v1/agent-session/ws?sessionId=<uuid>&wsToken=<token>
```

说明：

- `sessionId` 和 `wsToken` 必须来自 `POST /open/v1/sessions`
- `wsToken` 过期或无效时，连接会被拒绝

### 2. 发送消息

连接建立后，客户端直接发送纯文本消息即可。

示例：

```text
script_type=一句话剧本
script_content=一个被陷害的女律师重生回庭审前夜，反杀幕后黑手并逆袭成律所合伙人。
target_audience=女频
expected_episode_count=5
```

如果服务端返回的最终消息里带有交互按钮：

- `kind = send`: 直接把 `payload` 作为下一条 WebSocket 消息发送
- `kind = prefill`: 将 `payload` 预填到输入框，等待用户补充后再发送

示例：

```text
interaction_action=confirm
stateId=step1_input_parse
```

### 3. 接收事件

当前会透传两类事件：

- `message`: 最终消息对象
- `delta`: 流式增量

`message` 示例：

```json
{
  "version": "1.0",
  "eventType": "message",
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "sessionId": "upstream-session-id",
  "message": {
    "version": "1.0",
    "messageId": "msg-001",
    "instanceId": "00000000-0000-0000-0000-000000000000",
    "sessionId": "upstream-session-id",
    "sequence": 12,
    "role": "assistant",
    "content": "# 输入解析\n...",
    "thinkingContent": null,
    "pending": false,
    "interaction": {
      "version": "1.0",
      "type": "approval_request",
      "stateId": "step1_input_parse",
      "title": "请确认当前内容",
      "actions": [
        {
          "id": "confirm",
          "label": "确认并继续",
          "kind": "send",
          "payload": "interaction_action=confirm\nstateId=step1_input_parse"
        }
      ]
    },
    "emittedAt": "2026-03-10T03:05:00Z"
  },
  "emittedAt": "2026-03-10T03:05:00Z"
}
```

`delta` 示例：

```json
{
  "version": "1.0",
  "eventType": "delta",
  "instanceId": "00000000-0000-0000-0000-000000000000",
  "sessionId": "upstream-session-id",
  "delta": {
    "version": "1.0",
    "messageId": "msg-001",
    "instanceId": "00000000-0000-0000-0000-000000000000",
    "sessionId": "upstream-session-id",
    "sequence": 10,
    "role": "assistant",
    "channel": "content",
    "operation": "append",
    "chunk": "# 输入解析",
    "emittedAt": "2026-03-10T03:04:58Z"
  },
  "emittedAt": "2026-03-10T03:04:58Z"
}
```

说明：

- `channel = content`: 正文增量
- `channel = thinking`: 思考流增量
- `operation = append`: 追加内容
- `operation = clear`: 清空当前通道缓存

## 最小接入示例

```ts
const sessionResp = await fetch('/open/v1/sessions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-Open-App-Id': 'partner-web',
    'X-Open-App-Secret': 'your-secret',
  },
  body: JSON.stringify({
    instanceId: '00000000-0000-0000-0000-000000000000',
    agentId: 'mgc-novel-to-script',
    externalSessionKey: 'biz-session-001',
  }),
});

const session = await sessionResp.json();
const wsBase = location.origin.replace(/^http/i, 'ws');
const socket = new WebSocket(`${wsBase}${session.websocketPath}`);

socket.onopen = () => {
  socket.send([
    'script_type=一句话剧本',
    'script_content=一个被陷害的女律师重生回庭审前夜，反杀幕后黑手并逆袭成律所合伙人。',
    'target_audience=女频',
    'expected_episode_count=5',
  ].join('\n'));
};

socket.onmessage = (event) => {
  const frame = JSON.parse(event.data);
  if (frame.eventType === 'delta') {
    return;
  }
  if (frame.eventType === 'message') {
    return;
  }
};
```

## 错误与限制

- `400`: 参数缺失或格式错误
- `401`: `appId/appSecret` 不合法，或 `wsToken` 无效/过期
- `404`: 会话不存在，或不属于当前应用
- `409`: `externalSessionKey` 已绑定到其他目标，或会话已关闭

当前限制：

- 暂未提供纯 REST 的消息发送接口，消息收发必须走 WebSocket
- 外部接口不会返回 `eventType=debug` 调试事件
- 历史消息接口不保证包含全部实时 `delta` 增量
- 第三方系统只应依赖 `/open/v1/**`，不要耦合控制台内部 `/ops/v1/**` 接口
